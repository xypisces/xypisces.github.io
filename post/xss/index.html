必知必会--浏览器安全知识

发布于 2018-07-14 | 分类于 Web | 3分钟 | 564字数

前端涉及到的安全问题主要有二大块，xss攻击和csrf攻击，虽然前端发展至今已经有很多抵御这两种攻击的方式了，但也不是不能不了解的知识点

XSS

xss主要有三种：反射型，存储型，DOM-based

攻击过程

通过修改HTML节点或者执行JS代码来攻击网站

例如： <div>{{name}}</div>, 如果name的值是一段js代码，那么在页面上就会被执行

变成<div><script>alert("1")</script></div>,比如提交表单的时候进行执行，这种就叫做存储性攻击，因为每个看到这个表单内容都会触发这个攻击，可以用来获取网站用户cookies保存到自己的服务器当中

又或者在URl节点中嵌入JS代码

http://www.domain.com?name=<script>alert(1)</script>
<div>{{name}}</div>

输入上面URL的时候，页面就多一段可执行的js脚本，这中叫做反射型攻击

如何防御

  • 对输入输出的代码进行转义处理
  • 针对富文本可以采用添加白名单进行过滤处理，避免有危险的标签
  • CSP，在http头部添加content-security-policy进行控制（本质也是黑白名单控制）

CSRF

利用用户的登陆状态发起恶意请求

如何攻击

假设网站中有一个通过 Get 请求提交用户评论的接口，那么攻击者就可以在钓鱼网站中加入一个图片，图片的地址就是评论接口

<img src="http://www.domain.com/xxx?comment='attack'"/>

如果接口是 Post 提交的，需要用表单来提交接口

<form action="http://www.domain.com/xxx" id="CSRF" method="post">
    <input name="comment" value="attack" type="hidden">
</form>

如何防御

  • get请求无法修改数据
  • 不让第三方网站访问到cookie，比如sameSite（cookie不会跨域请求携带）
  • 阻止第三方网站请求接口， 比如 referer （验证请求来源）
  • 请求时附带信息，比如token（服务器下发，每次请求时携带进行验证）
